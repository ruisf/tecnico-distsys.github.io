Software

Para realizar os exercícios de laboratório e o projecto de SD e ES é necessário instalar um conjunto de ferramentas e servidores. Todo o software listado abaixo está disponível em Windows, Linux e Mac.

A maior parte das ferramentas necessita de variáveis de ambiente.
O procedimento para definir variáveis de ambiente depende do sistema operativo: Windows, Linux e Mac.

  1. Java Developer Kit, JDK 8u72 (que inclui o Java Runtime Environment, JRE)
    Para executar aplicações Java e applets, é necessário ter instalado o JRE.
    Para desenvolver aplicações Java ou appplets é necessário o JDK.
    • Instalar
    • Definir variável de ambiente JAVA_HOME
    • Acrescentar JAVA_HOME/bin à variável de ambiente PATH
    • Executar comando java -version para testar
  2. Apache Maven (preferência pela versão 3.3.3, que é a mesma que é utilizada internamente no Eclipse Mars.1)
    O Maven é uma ferramenta de linha de comando de gestão de todo o ciclo de vida de uma aplicação, incluindo a gestão de dependências de bibliotecas. Pode ser executado dentro do Eclipse mas também é aconselhável que esteja disponível na linha de comando.
    • Instalar
    • Definir variável de ambiente M2_HOME
    • Acrescentar M2_HOME/bin à PATH
    • Executar comando mvn --version para testar
  3. Git (preferência por versão maior ou igual a 2.x. Escolher a versão estável mais recente disponível para a plataforma)
    O Git é um sistema de controlo de versões distribuído. Esta instalação disponibiliza as ferramentas de linha de comando para interacção com repositórios Git.
    • Instalar
  4. Eclipse IDE for Java EE (4.5.1 Mars) Service Release 1
    Ambiente de desenvolvimento para programadores Java poderem criar programas Java EE e aplicações web, inclui um editor de texto com reconhecimento de código e ferramentas para Java EE, JPA, JSF, Ant, etc.
    • Instalar
      Atenção: se instalou o JDK 32-bit, deve instalar o Eclipse 32-bit;
      se instalou o JDK 64-bit, deve instalar o Eclipse 64-bit.
    • Definir variável de ambiente ECLIPSE_HOME
    • Plug-ins - para instalar cada plug-in ir a: Eclipse, Help, Install New Software..., Work with:
      preencher endereços 'Indicar' listados abaixo e escolher pacotes
      • Eclipse Mylyn connector for GitHub
        • Indicar: http://download.eclipse.org/releases/mars/
        • Seleccionar pacote dentro de 'Collaboration':
          • Eclipse GitHub integration with task focused interface
      • Eclipse plugin for FindBugs
        • Indicar: http://findbugs.cs.umd.edu/eclipse
        • Seleccionar pacote FindBugs
    • Conectores m2e
      Pode ser necessário instalar o plugin Maven2Eclipse antes através de: Help, Install New Software...
      • Abrir as preferências do eclipse e escolher -> Maven -> Discovery e clicar em "Open catalog".
      • Seleccionar:
        • m2e connector for jaxws-maven-plugin
        • m2e connector for maven-dependency-plugin
        • m2e connector for org.codehaus.mojo:jaxb2-maven-plugin
        • m2e-egit

Servidores

  • MySQL (preferência por uma versão da linha 5.6 mas não é um requisito estrito)
    • Definir variável de ambiente MYSQL_HOME
    • Acrescentar MYSQL_HOME/bin à PATH
    • Criar o utilizador de administração root:rootroot (utilizador:senha)
    • Instalar como service/daemon - com opção de auto-start
    • A instalação deverá incluir:
      • MySQL Workbench
      • MySQL Connector/J
    • Mais instruções:
      • Windows
      • OS X
      • Linux
  • JUDDI 3.3.2 (Servidor)
    Este pacote permite lançar um servidor UDDI para registo de serviços (UDDI registry)
    Instruções de instalação
    • Descomprimir para uma directoria local
    • Definir variável de ambiente CATALINA_HOME (...\juddi-distro-3.3.2\juddi-tomcat-3.3.2)
    • Acrescentar CATALINA_HOME/bin à variável de ambiente PATH
    • Num sistema de tipo Unix (Linux ou Mac), tornar os scripts de lançamento executáveis:
      chmod +x $CATALINA_HOME/bin/*.sh
    • Configurar jUDDI para escutar no porto 9090
      • Abrir o ficheiro '...\juddi-distro-3.3.2\juddi-tomcat-3.3.2\conf\server.xml' e alterar a linha 71 para:
          <Connector port="9090" protocol="HTTP/1.1"
    • Para lançar o servidor, basta executar:
      • startup.sh (Unix)
      • startup.bat (Windows)
    • Para confirmar funcionamento, aceder à página de índice do jUDDI, que dá também acesso à interface de administração
      http://localhost:9090/juddiv3/
